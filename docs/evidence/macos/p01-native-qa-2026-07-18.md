## Defects found and fixed during native QA

The pre-P01 installed bundle failed Person creation at the Tauri argument boundary. After that contract was corrected, compiled WebKit exposed another problem that the Chromium fixture did not: the asynchronous submit handler read `event.currentTarget` after awaiting the native command. WebKit cleared the event target, so the Person file was written but the UI reported failure and invited a dangerous retry.

The P01 implementation captures the form before the await, supplies a keyboard-submit fallback, and tests Enter submission. It also restores the platform meaning of the HTML `hidden` attribute so the pre-workspace warning is not announced after a workspace opens.

The exact-head Windows desktop job then found a second cross-platform contract defect: the Markdown adapter passed the host-rendered path `people\\malformed.md` into a shared structured Health finding, while every adapter fixture requires the logical workspace path `people/malformed.md`. Commit `33727f88a90eddd664dcaa9c2943387b1f9a9db2` converts relative path components to `/` at the filesystem anti-corruption boundary and falls back to the safe `people` scope instead of an absolute private path if relativisation fails. The unchanged desktop parity assertion remains the Windows regression gate.

The broader Windows workspace run then exposed an inverse fixture error: the default-workspace unit test imposed `/Users/example/Documents/Liaison RM` even though an OS-provided absolute Documents path must retain native separators. The production implementation was already correct. The test now constructs its expectation with `PathBuf::join`, preserving the distinction between native absolute filesystem paths and portable logical record paths.
